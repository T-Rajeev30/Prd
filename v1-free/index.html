02
Strategic Justification
The executive case — market gap, technical moat, and the rationale for investing in a custom-built platform.
🎯

The Vision: The Bloomberg of the Hobby

KLLCTBLS is not just a directory; it is a data-moat. By centralising fragmented show dates, shop locations, and sponsor intelligence, we are building the first institutional-grade infrastructure for the $10B+ trading card industry — the "Single Source of Truth" that everyone from casual collectors to deep-pocketed sponsors has been waiting for.

1. The Market Vacuum

The trading card hobby currently suffers from information asymmetry. Professional dealers have the data; casual collectors are left with table scraps.

  • ✗
    Scattered Show Data: Finding a local show requires checking 5+ legacy sites and 10+ Facebook groups.
  • ✗
    Disconnected Shops: Shop specialty (Pokemon vs Sports) is rarely indexed, causing wasted travel.
  • ✗
    Opaque Sponsorships: Brands have no clear way to track their ROI across multiple shows.

2. The Leo Multiplier

Industry Alpha A platform is only as good as its data. Leo's background as Marketing Director for the Dallas Card Show provides a direct pipeline to the most gated data in the industry.
  • Direct Lines: Leo has direct access to PSA, Heritage, Goldin, and NSCC organizers.
  • Growth Record: 10x social growth in 24 days — a proven content strategy baked into Phase 3.
  • Right Hand Model: India's rapid iteration speed combined with US-standard technical oversight.

3. Technical Alpha: Why Custom Code over Templates?

Tech Choice 01
Next.js 14 (App Router)
Enables Server-Side Rendering. Every event page indexed by Google instantly — massive SEO advantage over single-page apps.
Tech Choice 02
Supabase + pgvector
Not just storing text — preparing for Semantic Search in Phase 2. Collectors can ask natural-language questions and get intelligent results.
Tech Choice 03
Edge-First Scrapers
Scrapers run on Vercel Edge, monitoring 20+ sources nightly without expensive server costs. High speed, low overhead.

4. Development Path Comparison

Development Path Capabilities Long-term Risk
White-label / WP Template Static directory only. Zero AI scalability. No custom logic. High — must restart from zero for Phase 2.
Silicon Valley Agency Premium build, but massive overhead and corporate slowdown. Medium — high burn rate kills the startup.
Rajeev (Technical Partner) Bespoke architecture. Future-proof code. AI-ready from Day 1. Minimal — transparent ownership + lean build.
Strategic Conclusion By building the foundation correctly in Phase 1, we avoid significant future re-development costs. We are building the KLLCTBLS moat today so we can dominate the market tomorrow.

Security Architecture
How user data, API keys, and the admin panel are protected from day one.
Security Layers
Transport
HTTPS Everywhere
Automatic SSL via Vercel. All traffic encrypted. No exceptions.
Authentication
Supabase Auth + JWT
JWT tokens. Secure session management. Auto-expired tokens.
Database
Row Level Security
Postgres RLS. Public reads published data only. Admins see everything. Users see only their own data.
API Keys
Vercel Env Variables
Zero API keys in code. All in server-side environment variables. Never exposed to the browser.

Data Scraping — Legal Risk Assessment

Data Source Legal Risk Our Approach Fallback Plan
TCDB (Trading Card Database) High Review ToS before scraping. Use public data only. No login-gated content. Switch to manual entry + Leo's direct contacts.
Sports Collectors Digest Medium Scrape publicly listed show calendar. Attribute source. User submissions replace automated data.
Google Places API Low Official API with Leo's API key. Fully compliant. No fallback needed.
Card Ladder / ALT High Do NOT scrape. Their ToS explicitly prohibits it. Use eBay's official Browse API instead for pricing data.
PSA Pop Reports High Phase 2 only. Legal review required before any data collection. Partner with PSA directly via Leo's contacts — the preferred path.
Heritage / Goldin Auction Results Medium Scrape publicly available sold listings. No login bypass. Link to auction pages instead of storing data locally.
User Submissions Low Users voluntarily submit. ToS gives us permission to display. No risk — user-generated content.
Bottom Line for Leo Rajeev handles all the technical legal pages (Privacy Policy, ToS, disclaimers). Leo should consider forming an LLC and filing a trademark before Phase 2 monetization begins. For data scraping, we take a cautious approach: if a source is risky, we skip it and use Leo's direct industry contacts instead. No legal surprises.
