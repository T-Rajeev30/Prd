02
Strategic Justification: The Executive Case
A comprehensive analysis of the market gap, the technical moat, and the economic rationale for the $1,200 MVP investment.
🎯

The Vision: The Bloomberg of the Hobby

KLLCTBLS is not just a directory; it is a data-moat. By centralizing fragmented show dates, shop locations, and sponsor intelligence, we are building the first institutional-grade infrastructure for the $10B+ trading card industry. This is the "Single Source of Truth" that everyone from casual collectors to deep-pocketed sponsors has been waiting for.

1. The Market Vacuum (Current State)

The trading card hobby is currently suffering from "Information Asymmetry." Professional dealers have the data; casual collectors are left with table scraps. This fragmentation causes massive friction:

  • Scattered Show Data: Finding a local show requires checking 5+ legacy sites and 10+ Facebook groups.
  • Disconnected Shops: Shop specialty (Pokemon vs Sports) is rarely indexed, causing wasted travel for collectors.
  • Opaque Sponsorships: Brands have no clear way to track their ROI across multiple shows.

KLLCTBLS eliminates this friction, capturing the user's attention at the moment of discovery—the most valuable stage of the consumer funnel.

2. The "Leo Multiplier"

Industry Alpha

A platform is only as good as its data. Leo's background as Marketing Director for the Dallas Card Show provides a direct pipeline to the most gated data in the industry.

  • Direct Lines: Leo has direct access to PSA, Heritage, Goldin, and NSCC organizers.
  • Growth Record: 10x social growth in 24 days isn't luck; it's a proven content strategy that will be baked into Phase 3.
  • Right Hand Model: Collaboration with Leo's US-based software engineer ensures "Best of Both Worlds" development—India's rapid iteration speed combined with US-standard technical oversight.

3. Technical Alpha: Why Custom Code over Templates?

TECH CHOICE 01
Next.js 14 (App Router)
Enables Server-Side Rendering (SSR). This means every event page is indexed by Google instantly, giving us a massive SEO advantage over single-page apps.
TECH CHOICE 02
Supabase + pgvector
We aren't just storing text. We are preparing for Semantic Search in Phase 2, allowing collectors to ask "Find shows with high-end vintage" and get intelligent results.
TECH CHOICE 03
Edge-First Scrapers
Scrapers run on Vercel Edge, allowing us to monitor 20+ sources nightly without expensive server costs. High speed, low overhead.

Security Architecture
How user data, API keys, and the admin panel are protected from day one.
Security Layers
Transport
HTTPS Everywhere
Automatic SSL via Vercel. All traffic encrypted. No exceptions.
Authentication
Supabase Auth + JWT
JWT tokens. Secure session management. Auto-expired tokens.
Database
Row Level Security
Postgres RLS. Public can read published data only. Admins see everything. Users see only their own data.
API Keys
Vercel Env Variables
Zero API keys in code. All in server-side environment variables. Never exposed to browser.
Rate Limiting
Vercel Edge
AI chat: 20-50 req/user/hour. Submission form: 5/hour per IP. Prevents abuse.
SQL Injection
Drizzle ORM
All DB queries use parameterized statements. No raw SQL from user input.
What Data Is Stored About UsersEmail address only (for auth and alerts). Saved show/shop IDs. Topic preferences. No passwords stored in plain text — handled by Supabase Auth with bcrypt hashing. No payment data ever stored on KLLCTBLS servers — that goes directly to Stripe.
