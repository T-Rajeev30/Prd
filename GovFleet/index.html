Executive Summary

GovFleet is an end-to-end IoT-based GPS vehicle tracking and fleet management solution purpose-built for Indian government procurement. The product targets the massive, underserved government fleet segment — encompassing police vehicles, ambulances, municipal trucks, staff cars, state transport buses, PSU fleets, and defense vehicles — estimated at over 5 lakh vehicles across central and state government bodies.

The Problem

Government vehicles are largely untracked. No real-time visibility into vehicle location, driver behavior, fuel consumption, or maintenance schedules. This leads to unauthorized use, fuel theft, missed maintenance, inflated logbook entries, and zero accountability. Existing solutions are designed for private logistics — none are purpose-built for government procurement workflows.

The Solution

AIS 140-certified GPS hardware paired with a cloud-based SaaS platform featuring real-time tracking, geofencing, driver behavior scoring, fuel analytics, predictive maintenance, SOS/panic alerts, and automated compliance reporting. Designed for GeM listing and government procurement from Day 1.

Revenue Potential

At 10,000 vehicles in Year 1: ₹8-10 Cr revenue. At 50,000 vehicles by Year 3: ₹55+ Cr annually with 60%+ blended margins. Seed capital required: ₹13-15 Lakhs.

Section 03

Regulatory Landscape

The regulatory environment is the single biggest tailwind for GovFleet. Multiple overlapping mandates create compliance-driven demand for GPS tracking in government vehicles.

AIS 140 (Automotive Industry Standard 140)

Published by ARAI under MoRTH, AIS 140 mandates intelligent transportation system devices for all public transport and commercial vehicles.

  • GNSS-based real-time tracking with GPS + NavIC/IRNSS support
  • Emergency SOS/panic button hardwired into vehicle
  • Dual IP data transmission — simultaneously to government server and operator server
  • Minimum 4-hour battery backup on device
  • Tamper-proof enclosure with tamper detection alerts
  • 90-day on-device data storage capability
  • Integration with VAHAN national vehicle registry
  • GSM/GPRS communication module for data transmission
Compliance Deadline

All commercial vehicles registered before January 2025 must comply by October 31, 2025. Non-compliance results in fines, permit suspension, or operational halts.

Section 10

Product Architecture

System Overview — 4 Layers

Layer 1: Hardware

GPS device, SOS button, OBD connector, backup battery, tamper-proof enclosure. ESP32-S3, u-blox NEO-M8N, Quectel EC200U.

Layer 2: Communication

4G LTE / NB-IoT connectivity, MQTT protocol, edge processing, 32MB local buffer for offline operation.

Layer 3: Cloud Platform

AWS IoT Core, TimescaleDB (time-series), Node.js API, Redis (real-time cache), Kafka (stream processing), rule engine.

Layer 4: Applications

React.js web dashboard, React Native mobile app, PDF report engine, multi-channel alert system (push, SMS, email).

Data Flow

  • GPS device collects location + speed + engine data every 10 seconds
  • Data transmitted via MQTT over 4G to AWS IoT Core (primary) and VAHAN server (secondary) for AIS 140 dual-IP compliance
  • Kafka normalizes data → TimescaleDB (historical) + Redis (real-time cache)
  • Rule engine evaluates geofence breaches, overspeed, SOS, idle time, fuel anomalies → fires alerts
  • Dashboard reads from Redis (live) and TimescaleDB (analytics)
  • Report generator creates scheduled PDF/Excel reports delivered to officers

Section 14

User Stories & Acceptance Criteria

US-001
"As a Transport Officer, I want to see all my vehicles on a live map so that I can know which vehicles are on route and which are idle."
Acceptance Criteria
Map loads within 3 seconds. Vehicle positions update every 10 seconds. Color coding: Green=moving, Yellow=idle >5min, Red=stopped >30min, Gray=offline. Click on vehicle shows speed, driver name, last update time.
US-002
"As a Transport Officer, I want to receive an instant alert when a vehicle exits its assigned geofence so that I can take immediate action on unauthorized use."
Acceptance Criteria
Alert within 30 seconds of boundary crossing. Push notification + SMS to designated officer. Alert includes vehicle number, driver, location, time. Option to immobilize vehicle remotely.
US-003
"As a Commissioner, I want a monthly fleet utilization report auto-delivered to my email so that I can review performance in monthly meetings."
Acceptance Criteria
Auto-generated on 1st of every month. Includes: total km per vehicle, fuel consumed, idle %, trip count, maintenance alerts, top 5 overspeed incidents. PDF format, branded with department logo.
US-004
"As a Driver, I want to press an SOS button during an emergency so that my location is immediately shared with the control room."
Acceptance Criteria
SOS triggers within 2 seconds. Alert reaches control room with audio/visual alarm. Location shared every 5 seconds during SOS mode. Requires manual acknowledgment to deactivate.
US-005
"As an IT Administrator, I want to manage user roles and department hierarchy so that each department sees only their vehicles."
Acceptance Criteria
Create department tree (Ministry → Dept → Division → Section). Assign vehicles to nodes. Users see only their node and below. Super-admin sees all. Audit log for every permission change.
US-006
"As a Finance Officer, I want fuel consumption reports per vehicle so that I can identify fuel theft and reduce fleet costs."
Acceptance Criteria
Daily fuel consumption in liters and cost. Km/liter trend per vehicle. Alert when consumption deviates >20% from baseline. Fill/drain events detected. Monthly cost summary exportable to Excel.
