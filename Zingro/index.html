1. Executive Summary

ZINGRO is a mobile-first food platform connecting customers with restaurants and home-based cooks, with a core emphasis on subscription-based daily meal plans. It addresses three unmet needs simultaneously: affordable home-style food, a marketplace for homemakers to monetize cooking skills, and reliable meal subscriptions for everyday users.

Mission: To make home-style, affordable food accessible to everyone — daily — while empowering homemakers to build sustainable income from their kitchens.

The Problem

  • Existing platforms (Zomato, Swiggy) are restaurant-centric and expensive for daily use
  • Homemakers with culinary skills have no structured platform to sell meals
  • Working professionals and students lack access to affordable, reliable daily meal subscriptions
  • No existing platform combines restaurant ordering + homemade food + subscriptions in one app

The Solution

  • Three-sided marketplace: Customers, Homemakers, Delivery Partners
  • Dual ordering modes: one-time orders and recurring subscriptions
  • Homemade food marketplace with trust signals (hygiene badges, ratings)
  • Affordable-first pricing with transparent costs shown before checkout

9. User Stories & Acceptance Criteria

US-C01: Subscribe to a Homemaker Meal Plan

As a customer, I want to subscribe to a homemaker's meal plan so I don't need to order every day.

Acceptance Criteria:

  • Customer selects meal type (lunch/dinner) and plan duration (weekly/monthly)
  • Start date selectable; same-day only if placed 2+ hours before delivery slot
  • Total cost and per-meal cost displayed before confirmation
  • Subscription appears in dashboard immediately after payment success
  • System auto-generates daily orders at 6 AM for each active subscription
  • Paused days are skipped without extending plan duration

US-C02: Track an Order

As a customer, I want to see order status so I know when my food will arrive.

Acceptance Criteria:

  • 5-step status bar: Placed → Accepted → Preparing → Out for Delivery → Delivered
  • Push notification sent at each status change
  • Dunzo tracking link visible when status is Out for Delivery
  • Estimated delivery time shown and updated

US-C03: Pause a Subscription

As a customer, I want to pause my subscription for specific days so I don't pay for food I won't receive.

Acceptance Criteria:

  • Pause must be set at least 2 hours before scheduled delivery time
  • Paused days: no order generated, no charge, no plan extension
  • Calendar view shows paused vs active days
  • Maximum 5 pause days per month per plan

US-H01: Add a Meal

As a homemaker, I want to add meals with name, price, and availability so customers can discover and order from me.

Acceptance Criteria:

  • Homemaker adds: meal name, description, photo, price (single/weekly/monthly), veg/non-veg tag
  • Availability toggle per meal (active/inactive)
  • Meal goes live on customer app within 5 minutes of save

US-H02: Accept or Reject an Order

As a homemaker, I want to accept or reject incoming orders so I only take on what I can handle.

Acceptance Criteria:

  • New order notification with 10-minute acceptance window
  • Auto-cancel and customer refund if not accepted in 10 minutes
  • Rejection requires selecting a reason from predefined list

US-D01: View and Accept a Delivery

As a delivery partner, I want to see order details before accepting so I can evaluate pickup/drop distance.

Acceptance Criteria:

  • Order card shows: pickup location, drop location, distance, order type
  • Accept/Reject buttons visible; auto-assigned via Dunzo if not accepted in 5 min
  • Navigation launches on single tap after acceptance

13. Technical Architecture

13.1 System Overview

Three Flutter apps (Customer, Homemaker, Delivery Partner) communicate via a single Node.js/Express REST API backed by PostgreSQL on Supabase. Dunzo handles delivery logistics. Razorpay handles payments. Firebase handles push notifications.

13.2 Flutter App Architecture

Pattern: Feature-Based Architecture + Riverpod State Management

Layer Purpose Examples
core/ Global utilities, theme, routing, API client AppTheme, ApiService, AppRouter
shared/ Cross-feature models and providers UserModel, AuthProvider, OrderModel
features/ Isolated self-contained feature modules cart/, order/, homemaker/, subscription/
app/ App configuration and entry wiring app.dart, routes.dart, main.dart

13.3 Riverpod State Management

Riverpod Provider Type Use Case
Provider Simple dependencies (theme, config, API client)
FutureProvider API data fetching (homemaker list, meal details, order status)
StateNotifierProvider Complex mutable state (cart contents, subscriptions, auth session)

13.4 Feature Internal Structure

Each feature folder follows this pattern:

  • data/ — API calls, repository classes
  • domain/ — Business logic, use cases (optional in Phase 1)
  • presentation/screens/ — UI screens
  • presentation/widgets/ — Reusable UI components for this feature
  • presentation/providers/ — Riverpod providers for this feature's state

13.5 Order Lifecycle

CREATED → PENDING_ACCEPTANCE → ACCEPTED → PREPARING → READY → OUT_FOR_DELIVERY → DELIVERED

  • If homemaker does not accept in 10 min: Auto-cancel, customer refunded via Razorpay
  • If Dunzo delivery fails: Retry once; if second failure, escalate to ops and refund

13.6 Background Jobs

Job Schedule Action
Subscription Order Generator Daily at 6:00 AM For each active subscription, create order for the day; skip paused dates
Order Timeout Handler Every 5 minutes Cancel orders in PENDING_ACCEPTANCE over 10 min; trigger Razorpay refund
Subscription Renewal Check Daily at midnight Flag subscriptions ending within 3 days; send renewal push notification

17. Homemaker Onboarding & Operations

17.1 Onboarding Flow

  1. Homemaker downloads the Homemaker App and registers with phone number (OTP verification)
  2. Completes profile: name, kitchen location (area-level), cooking capacity per day
  3. Uploads government ID (Aadhaar) and signs food safety self-declaration
  4. Adds at least 1 meal with photo, price, and availability schedule
  5. ZINGRO ops team reviews listing within 24 hours (manual check in Phase 1)
  6. Account activated; homemaker goes live on customer app

17.2 Verification Requirements

Requirement Mandatory? Notes
Government ID (Aadhaar/PAN) Yes For identity verification and payouts
Kitchen photo Yes Reviewed by ops team; basic hygiene check
FSSAI Basic Registration Recommended Shown as hygiene badge; increases order conversion
Phone number (OTP verified) Yes At registration
Bank account for payouts Yes Required before first payout is released

17.3 Dispute Resolution

Scenario Resolution SLA
Wrong item delivered Customer raises ticket; auto-refund if homemaker confirms error 2 hours
Food quality complaint Ops team reviews; partial/full refund at discretion; homemaker notified 24 hours
Order not delivered Check Dunzo tracking; redeliver or full refund 4 hours
Homemaker cancels accepted order Customer auto-refunded; homemaker strike logged (3 strikes = suspension review) Immediate
Payment not received by homemaker Finance team reconciliation; manual payout if needed 48 hours

17.4 Refund and Cancellation Policy

Customer cancellations — one-time orders:

  • Before homemaker accepts: Full refund, no penalty
  • After homemaker accepts but before preparation: 50% refund
  • After preparation begins: No refund (food has been prepared)

Subscription cancellations:

  • Cancel anytime; no refund for current billing period; next period not charged
  • Pause: up to 5 days per month; no refund or plan extension for paused days

20. Go-to-Market Strategy

20.1 Launch City: Bangalore First

Bangalore is recommended as the Phase 1 launch city over Pune for three reasons:

  • 3x larger informal tiffin market (Rs 600 Cr vs Rs 200 Cr)
  • Higher density of target customers (2M IT workforce vs 700K in Pune)
  • Dunzo has full city coverage in Bangalore; coverage in Pune is more limited
  • Migrant working population is highest — strongest home-food desire

Pune becomes Phase 2 city expansion at Month 3.

20.2 Pre-Launch Checklist (Week 1–2 = MVP Build)

  • Field ops team manually contacts homemakers in HSR Layout, Koramangala, Whitefield via WhatsApp, local Facebook groups, housing society contacts
  • Register Razorpay and Dunzo business accounts — do this Day 1, not Day 8
  • Set up basic customer support: WhatsApp Business number for issue resolution
  • Prepare homemaker onboarding kit: short video guide, FAQ document, referral incentive (Rs 500 bonus for first 5 orders)

20.3 Launch Phase (Week 3–4)

  • Invite-only beta with 100 users (employees, friends, early sign-ups)
  • First free week of subscription for first 200 users who subscribe
  • Referral program: Rs 50 credit per successful referral
  • Instagram Reels featuring homemakers cooking (authentic, no studio production)
  • Post in WhatsApp/Facebook tiffin groups in Koramangala, HSR Layout, Whitefield
